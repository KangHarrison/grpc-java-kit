# 服务注册和发现

## 演变趋势

服务的注册和发现是微服务软件架构中重要的需求之一，它承担着服务通讯过程中最重要的功能。 从以 Apache Dubbo 和 Spring Cloud 为代表的集成微服务开发框架时代，到当下流行的 Kubernetes
云原生时代，服务注册和发现的解决方案和落地实现，也随着时代浪潮经历过几次重要演变，下面就来具体展开说说。

### 服务发现的粒度

基于接口 -> 基于应用

### 服务发现的协议

DNS -> 注册中心 -> DNS

### CAP 的选择

CP -> AP
